## 1. 概要

既存の `capture-form.tsx` を案Aレイアウトへ全面置き換えする。  
ルートは `/`（`WebApp/src/app/page.tsx`）のまま。  
プロトタイプ（`/prototypes/capture-ui?variant=single-flow`）を実装ベースとして使う。

**コンセプト**: 撮影・座標・入力・送信が1画面内に同居し、スクロールを減らす。スマホ（390px幅）で快適に動くことを最優先とする。

---

## 2. 廃止・縮小する現行要素

| 現行要素 | 対応 |
|---|---|
| 担当場所 (location) | **廃止** — 今回の運用では不要 |
| 端末名 (deviceLabel) | **廃止** — 今回の運用では不要 |
| カメラ診断パネル | **廃止** — 常時表示は邪魔。問題時はコンソールで確認 |
| FILE FALLBACK 大枠 | **縮小** — 「画像を選ぶ」ボタンに統合 |
| メモ欄 (note) | **廃止** — 運用上不要 |
| シール色 (genre, 現行) | **置き換え** → 新ジャンル選択に |
| 地図エリアテキスト (mapArea) | **置き換え** → 緯度経度 + 年代に |

## 4. 各セクション仕様

### 4.1 ヘッダー

- `HCD CAPTURE HUB`（display-font, accent-strong色）
- タイトル: `付箋を撮って送る`
- 説明文: `送信完了番号が出たら次へ進めます。`（短く）
- スタイル: 現行踏襲（太線・クリーム背景・黄色アクセント）

---

### 4.2 担当者バッジ

**デフォルト表示（折りたたみ）**:
```
担当: 担当者  [変更]
```

**展開状態**:
```
担当者名 [テキスト入力]
         [保存して閉じる]
```

**実装詳細**:
- `localStorage` キー: `hcd.operatorName`（既存流用）
- 初回起動時（空の場合）は自動で展開して入力を促す
- `settingsLoaded` パターンで SSR ハイドレーション不一致を回避（既存踏襲）
- バリデーション: 担当者名が空のとき送信ボタンを `disabled`

---

### 4.3 カメラ/画像枠

**カメラ枠**:
- `aspect-[4/3]`（横長、現行の `3/4` 縦長から変更）
- カメラ起動前: 黒背景 + `CAMERA` 表示
- ライブビュー中: `<video>` 全面表示 + ガイド枠オーバーレイ
- 撮影後: クロップ済みプレビュー表示

**ガイド枠**:
- 現行踏襲: `aspect-[5/3]` 幅88%、コーナーマーカー付き
- カメラ起動中のみ表示

**操作ボタン行**（2等分グリッド）:
- 左: `画像を選ぶ`（白、角丸、`min-h-[72px]`）
  - `<input type="file" accept="image/*" capture="environment">` をラップ
  - ファイル選択後は自動クロップ処理（現行の `createGuidedCrop` 移植不可 → 画像ファイルそのまま使用）
- 右: `シャッター` / `撮り直す`（黄色ピル形、`min-h-[72px]`、`rounded-full`）
  - カメラ未起動時: タップで `startCamera()` → 起動後シャッター待機
  - カメラ起動中: タップで撮影（`captureFromCamera()`）
  - 撮影済み時: `撮り直す` に表示切替、タップでリセット

**カメラエラー**:
- エラー発生時のみ表示（常時表示しない）
- `<div>` で `text-[var(--danger)]` で1行表示

**canvas**: `hidden`（現行踏襲）

---

### 4.4 座標ミニマップ

プロトタイプの `MiniMapPicker` をそのまま移植。本番実装として以下を維持:

**年代タブ（フォルダ型）**:
- `1960 / 1970 / 1980 / 2005 / 2025`
- 5列グリッド、選択タブが白背景・未選択が薄背景
- タブ下辺の border を消して地図と密着（`border-b-0`）
- タップで年代切替 → mapLocked をリセット

**地図キャンバス**:
- `aspect-[3/2]`、`rounded-md rounded-t-none`（タブとの接続）
- `overflow-hidden`、`touch-none`（スクロール干渉防止）
- キャンバス実寸から動的にドラッグ範囲計算（`getMapDragBounds`）
- キャンバスは `-inset-32` で拡張（端まで到達可能に）
- PointerEvent でドラッグ（`setPointerCapture` 使用）

**クロスヘア**:
- 画面中央固定（`pointer-events-none absolute left-1/2 top-1/2`）
- 上下左右 4本の線（3px幅、20px長、`var(--line)` 色）
- 中心点: `h-1.5 w-1.5` 丸、`var(--accent-strong)` 色、影なし
- ラジアルグラデーションの薄暗い輪（中心の視認性向上）

**緯度経度表示**:
- 地図左下: `35.xxxxxx, 139.xxxxxx`（`getLatLngFromOffset` による計算）
- ヘッダー行にも年代/緯度経度バッジを表示

**座標指定ボタン**:
- 地図ヘッダー右端に配置（送信バーと重ならない高さ）
- `座標を指定` → クリックで `mapLocked = true`、表示が `指定しなおす` に変化
- `指定しなおす` → クリックで `mapLocked = false`、再ドラッグ可能
- ロック中: 地図キャンバス `opacity-80`

**状態メッセージ**:
- 未指定: `地図を動かして、中央ピンの位置で指定します。`
- 指定済み: `座標を固定中。指定しなおすと地図を動かせます。`

**座標計算（仮実装）**:
```ts
// 年代ごとに原点をずらした仮緯度経度
const baseLat = 35.8321 + eraIndex * 0.00018;
const baseLng = 139.9552 + eraIndex * 0.00022;
lat = baseLat - offset.y * 0.000012;
lng = baseLng - offset.x * 0.000012;
```
> **注意**: 本座標は仮実装。実際の地図データと対応させる作業は後続フェーズ。

### 4.5 書いた人の名前

```
書いた人の名前
[テキスト入力 placeholder="例: 佐藤 / ニックネーム可"]
```

- フォームフィールド: `visitorName`（`formData.set("visitorName", ...)` で送信）
- 任意入力（空でも送信可能）
- 既存の `nickname` フィールドに対応（APIへのキーは `nickname` のまま）

---

### 4.6 ジャンル選択

```
ジャンル
[恋愛] [友情] [学業]
[部活] [行事] [上記以外]
```

- 3列 × 2行グリッド（`grid-cols-3`）
- 選択済み: `bg-[var(--accent)]`、未選択: `bg-white`
- `aria-pressed` 付与
- タップ即反応（`onPointerDown` + `onClick`）
- デフォルト選択: なし（未選択を許容）または `友情`（要確認）
- API送信値: 選択値そのまま（`恋愛` / `友情` / `学業` / `部活` / `行事` / `上記以外`）

---

### 4.7 下部固定送信バー

```
sticky bottom-3
┌─────────────────────────┐
│  [X1へ送信]             │ ← 黄色ボタン、全幅
│  送信済み: 3             │ ← 右寄せカウンター
│  受付番号 xxxx           │ ← 送信成功時
└─────────────────────────┘
```

- `sticky bottom-3 z-10`（フローティング、座標モジュールの上）
- `X1へ送信` は `disabled` 条件: 画像なし OR 担当者名空
- 送信済みカウント: `localStorage["hcd.sentCount"]`

## 5. API変更

### 5.1 FormDataフィールド変更

| フィールド | 変更内容 |
|---|---|
| `operatorName` | 変更なし |
| `location` | **廃止**（APIは受け取るが空文字で送信） |
| `deviceLabel` | **廃止**（APIは受け取るが空文字で送信） |
| `nickname` | `visitorName`（書いた人の名前）として使用。キーは既存 `nickname` のまま |
| `genre` | 新ジャンル（恋愛/友情/学業/部活/行事/上記以外）をそのまま送信 |
| `mapArea` | **廃止** → 緯度経度で置換 |
| `note` | **廃止**（APIは受け取るが空文字で送信） |
| `latitude` | **新規追加**: 緯度（文字列、小数点6桁） |
| `longitude` | **新規追加**: 経度（文字列、小数点6桁） |
| `era` | **新規追加**: 年代（`1960`/`1970`/`1980`/`2005`/`2025`） |

### 5.2 `CaptureRecord` 型変更（`records.ts`）

```ts
// memory フィールドに追加
memory: {
  nickname: string;     // 書いた人の名前（既存）
  genre: string;        // ジャンル（新: 恋愛/友情/学業/部活/行事/上記以外）
  era: string;          // 年代（新: 1960/1970/1980/2005/2025）
  latitude: string;     // 緯度（新: 35.xxxxxx）
  longitude: string;    // 経度（新: 139.xxxxxx）
  mapArea: string;      // 廃止予定（後方互換で残す）
  note: string;         // 廃止予定（後方互換で残す）
}
```

### 5.3 APIルート変更（`route.ts`）

`/api/captures POST` で `era`, `latitude`, `longitude` を `memory` に追加。  
後方互換のため `mapArea` / `note` / `location` / `deviceLabel` は空文字として残す。
